## 1. Введение

**Основные понятия:** контейнеризация, оркестрация, под (Pod), сервис (Service), деплоймент (Deployment), репликасет (ReplicaSet).

**Преимущества K8s:** автомасштабирование, самовосстановление, rolling updates, управление конфигурациями, балансировка нагрузки.

**Сценарии:** микросервисы, CI/CD, обработка пиковых нагрузок, мультиоблачные развёртывания.

## 2. Архитектура

| Компонент master | Роль |
|----------------|------|
| API Server | Принимает запросы (kubectl) |
| etcd | Хранилище состояния |
| Scheduler | Назначает поды на узлы |
| Controller Manager | Управляет контроллерами |

| Компонент worker | Роль |
|----------------|------|
| kubelet | Запускает поды |
| kube-proxy | Сетевая маршрутизация |
| container runtime | containerd |
